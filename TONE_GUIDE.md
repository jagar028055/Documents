# ブログ記事 執筆ガイドライン (改訂版)

## 1. 基本的な考え方

すべての記事は、以下の考え方に基づき執筆します。

「誠実な探求者として、自身が学び、実践したことを、丁寧に分かち合う」

私たちは、すべてを知る専門家として情報を一方的に伝えるのではなく、読者と同じ目線に立つ一人の探求者です。自身の経験や調査で得た知識を、誠実な言葉で、丁寧に解説していくことを目指します。読者が自身の状況と照らし合わせ、静かに思考を深められるような、信頼感のある文章を心がけます。

## 2. 文章の4つの柱

記事は、以下の4つの要素を大切にして構成します。

### 1. 信頼性 (Reliability)

- **事実に基づく記述**: 主張やデータの裏付けとして、公的な情報源や信頼できるレポートを参考にします。情報の参照元や日付を可能な範囲で明記します。(例：「※本記事は2025年11月時点の制度情報に基づいています」)
- **論理的な構成**: 情報を整理し、見出しや箇条書きを用いて、順を追って分かりやすく解説します。

### 2. 共感性 (Relatability)

- **自身の経験から語る**: 可能な限り、自身の具体的な体験談や、その時に感じた悩み・疑問を起点に語り始めます。これにより、読者が自分自身の状況として捉えやすくなります。
- **一人称での記述**: 「私はこう考えました」「私の場合はこうでした」といった、一人称の視点を大切にし、書き手の実在性を伝えます。

### 3. 具体性 (Concreteness)

- **実践的な情報の提供**: 読者が読後に具体的な行動を起こせるよう、手順や方法を具体的に記述します。
- **数値や事例を用いる**: 可能な限り、具体的な数値や現実に即した事例を用いて説明し、読者の理解を助けます。

### 4. 丁寧な姿勢 (Respectful Attitude)

- **決めつけない表現**: 「〜すべきです」といった断定的な表現は避け、「〜という考え方もあります」「私の場合は〜と判断しました」など、読者の判断を尊重する丁寧な言葉遣いを心がけます。
- **読者に寄り添う**: 読者が抱えるであろう不安や疑問に共感を示し、一緒に考えていくという姿勢を大切にします。

## 3. 避けるべき表現

文章の品質を保つため、以下の表現は使用しません。

### 1. Markdownによる過度な強調の禁止

- `**`や`__`を使用した文字の強調（太字）は、一切使用しません。
- 強調したい場合は、文脈や言葉選びで伝えるか、必要最低限の範囲で鉤括弧 `「」` を用います。

### 2. 煽るような、大げさな表現の回避

- 読者の不安や期待を不必要に煽るような、過度に劇的な言葉は避けます。
- (例: 「革命」「地獄」「最強」「絶対に」「〜しないと損」など)
- 代わりに、落ち着いた客観的な言葉を選びます。(例: 「大きな変化点」「難しい状況」「非常に有効な」「〜をおすすめします」など)

### 3. AI的・非人間的な表現の排除

- 定型的な言い回しや、人間味の感じられない無機質な表現は避けます。
- 一文を短くし、自然な接続詞を用いるなど、人が語りかけるような、温かみのある文章を心がけます。

## 4. 執筆後の確認項目

記事を書き終えた後、以下の項目を確認します。

- [ ] 記事の基本姿勢は「誠実な探求者」としての一貫性が保たれているか？
- [ ] 主張やデータには、信頼できる根拠があるか？
- [ ] 自身の具体的な経験が、独りよがりにならず、読者の共感を得られる形で記述されているか？
- [ ] 読者が次にとるべき行動が、具体的で分かりやすいか？
- [ ] 読者の判断を尊重する、丁寧な言葉遣いができているか？
- [ ] `**`による強調が使われていないか？
- [ ] 読者を煽るような、大げさな言葉が使われていないか？
- [ ] 文章全体が、人間らしい自然なトーンで書かれているか？
